# 小铁 UI 样式指南 v1.0

## 1. 设计目标

- 风格：现代简约、低噪声、高信息密度。
- 参考：Claude 的柔和层级与 OpenCode 的高对比工作区。
- 核心原则：可读优先、操作闭环、状态可感知、错误可恢复。

## 2. 色彩体系

### 2.1 语义色板

| 语义 | Token | 用途 |
|---|---|---|
| 主色 | `$primary` | 主按钮、焦点态、关键强调 |
| 辅助色 | `$secondary` | 次要标签、辅助信息 |
| 成功 | `$success` | 成功提示、完成状态 |
| 警告 | `$warning` | 风险操作、处理中 |
| 错误 | `$error` | 错误提示、失败状态 |
| 信息 | `$info` | 系统提示、引导消息 |
| 背景 | `$background` | 全局底色 |
| 面板 | `$surface` | 卡片/输入区/弹层 |
| 边框 | `$border`/`$border-active` | 常态/焦点分隔 |
| 文本 | `$text`/`$text-muted` | 主文本/次级文本 |

### 2.2 对比度策略

- 主文本与背景对比度 >= 4.5:1。
- 小字号提示与背景对比度 >= 4.5:1。
- 焦点边框与邻近背景对比度 >= 3:1。

## 3. 字体与排版层级

| 层级 | 场景 | 样式建议 |
|---|---|---|
| L1 | 页面标题/弹层标题 | Bold + 主色 |
| L2 | 分组标题/状态标签 | Semibold |
| L3 | 正文消息 | Regular |
| L4 | 辅助说明/快捷键提示 | Muted + Dim |

- 行长控制：消息正文推荐 60-90 字符宽。
- 行距策略：信息区紧凑、说明区留白更大。

## 4. 视觉层级规范

- 一级区域：消息主区（最强视觉权重）。
- 二级区域：输入区、状态行（高频反馈）。
- 三级区域：会话侧栏、快捷提示（导航辅助）。
- 异常状态：错误与警告使用独立边框颜色 + Toast 反馈。

## 5. 动效与反馈

- 处理态：思考指示器 + 输入区禁用占位文案。
- 操作成功：轻量 Toast，默认 3 秒自动关闭。
- 错误反馈：错误内容 + 下一步提示（例如“可用 /help 查看命令”）。

## 6. 响应式规则

- 宽屏（>=130）：显示侧栏 + 完整状态行。
- 中屏（110~129）：压缩侧栏宽度。
- 窄屏（<110）：自动隐藏侧栏，聚焦消息与输入主流程。

## 7. 无障碍规则（WCAG 2.1 对齐）

- 全链路键盘可达：命令面板、模型选择、主题选择、会话切换。
- 焦点可见：可聚焦项提供高对比边框。
- 状态可理解：成功/警告/错误同时通过文本表达，不只依赖颜色。

## 8. 组件使用约束

- ChatMessage：必须带角色标识与时间戳。
- Editor：处理态必须禁用输入并可见反馈。
- SessionItem：支持点击和 Enter 触发。
- StatusLine：展示模型、token、会话、模式、主题。

## 9. 上线一致性检查

- 主题切换后颜色 token 生效。
- 侧栏自适应符合断点规则。
- 快捷键与帮助文档一致。
- 异常流程有明确提示与恢复路径。
