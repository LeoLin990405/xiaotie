# 基于 MiniMax 的仿制与学习计划（工程实践版）

## 1. 目标与边界

本计划目标不是复制产品外观，而是通过“能力拆解 + 工程复刻”掌握现代 Agent 的关键原理与实现方法。

- 学习目标：掌握 Long-Horizon 任务执行、多模态协同、MCP 扩展、成本与稳定性平衡
- 工程目标：在小铁中构建可验证的 MiniMax 风格能力子集
- 合规边界：仅基于公开资料与官方接口，不逆向私有实现，不复用受限资产

## 2. MiniMax 能力基线（公开信息抽象）

基于官方文档与公开介绍，可抽象出以下能力支柱：

- 通用对话与工具调用（OpenAI/Anthropic 兼容接入）
- 多模态输入理解与多媒体生成能力接入
- MCP 生态扩展（内置与第三方服务）
- 长程任务（Long Horizon）拆解与多步交付
- 多模型协同（能力与成本平衡）

## 3. 逆向分析框架（可执行）

### 3.1 产品能力逆向

- 输入类型：文本、文档、图片、音频、视频
- 输出类型：文本、图像、音频、视频、可运行产物
- 任务形态：问答、编程、信息整合、自动化流程

### 3.2 技术栈逆向

- 模型接入方式：OpenAI 兼容 + Anthropic 兼容
- 任务执行方式：多步规划 + 工具/MCP 调用 + 结果整合
- 运行策略：多模型路由、成本控制、失败回退

### 3.3 工程机制逆向

- 观测：步骤级事件、工具耗时、错误率、队列指标
- 安全：高风险审批、敏感信息拦截、审计字段
- 质量：任务级评测、回归集、灰度机制

## 4. 仿制实现总体架构

将仿制目标映射为 6 个可落地引擎：

1. **能力路由引擎**：按任务/模态/成本选择模型与工具
2. **规划执行引擎**：支持任务分解、并行节点与重规划
3. **记忆检索引擎**：多层记忆 + 语义检索 + 历史压缩
4. **多模态网关引擎**：输入标准化与输出聚合
5. **工具与 MCP 引擎**：统一协议、安全、审计、限流
6. **评测观测引擎**：质量/性能/成本三维评测

## 5. 分阶段路线图（12 周）

## 阶段 A：认知建模与能力对齐（第 1-2 周）

- 输出：
  - MiniMax 能力映射矩阵（能力 -> 模块 -> 接口）
  - 小铁差距矩阵（现状 -> 缺口 -> 优先级）
  - 基线任务集（20 个）

## 阶段 B：核心执行链复刻（第 3-5 周）

- 目标：
  - 实现 Plan-Act-Observe-Reflect 主链路
  - 接入任务图与重规划
- 输出：
  - `Planner`、`Executor`、`Reflector` 协议与实现
  - 执行报告结构（步骤、耗时、失败原因）

## 阶段 C：多模态与 MCP 复刻（第 6-8 周）

- 目标：
  - 实现多模态消息协议与路由
  - 对接至少 3 个 MCP 场景（代码仓库、协作、设计）
- 输出：
  - `MultiModalMessage` 协议
  - MCP 安全网关（权限 + 审计）

## 阶段 D：记忆与自主决策复刻（第 9-10 周）

- 目标：
  - 记忆写入、召回、压缩进入主链路
  - 决策策略支持规则/评分/反馈切换
- 输出：
  - 记忆命中率报表
  - 策略切换日志与收益分析

## 阶段 E：验证与工程化固化（第 11-12 周）

- 目标：
  - 完成评测体系与发布门禁
  - 输出学习总结与技术复盘
- 输出：
  - Benchmark 报告
  - 稳定性与成本报告
  - 技术白皮书

## 6. 关键算法实现清单

### 6.1 任务规划与重规划

- 任务分解：目标 -> 子任务 DAG
- 调度策略：关键路径优先 + 并行执行
- 重规划触发：工具失败、上下文不足、风险拒绝

### 6.2 工具选择与执行

- 工具选择打分：可用性、历史成功率、成本、时延
- 执行控制：超时、重试、熔断、降级
- 审计闭环：入参与结果摘要、调用来源、风险等级

### 6.3 记忆检索与上下文编排

- 混合检索：关键词 + 向量相似度 + 时间衰减
- 记忆压缩：摘要写回 + 噪声剔除
- 上下文预算：按任务阶段动态分配 token

### 6.4 自主决策

- 动作候选生成
- 评分函数：任务收益 - 风险成本 - 资源成本
- 反馈学习：将成功/失败信号写入策略参数

## 8. 测试验证方案

## 8.1 功能正确性

- 单元测试：协议实现、策略分支、异常路径
- 集成测试：规划-执行-重规划闭环
- E2E：长程任务 20 条基准集

## 8.2 性能与成本

- 指标：
  - 任务完成率
  - 首响应时延、P95 时延
  - 单任务 token 与工具调用成本
  - 失败恢复成功率

## 8.3 稳定性与安全

- 压测：并发会话、长会话、MCP 密集调用
- 安全：高风险误放行率、敏感输出泄露率
- 审计：关键调用字段覆盖率

  - 平台工程：运行时与工具层

## 11. 风险与应对

- 外部 API 变更：抽象模型网关，隔离 provider 细节
- 多模态成本上升：能力路由优先低成本链路
- 任务不稳定：重规划 + 降级执行 + 审计回放
- 需求蔓延：每阶段设“冻结范围”和验收门禁
