# TUI 交互式工具交付说明（2026-03-06）

## 1. 产品定位

本次交付为一个基于终端文本界面（TUI）的交互式工具，设计理念对齐 Claude 与 Opencode：

- 保持终端原生、低干扰、高密度信息呈现
- 优先键盘操作路径，减少鼠标依赖
- 强调可引导、低误触、可恢复的交互模型

## 2. 核心能力

- 分区布局
  - 消息主区（对话与工具输出）
  - 侧栏会话区（上下文切换）
  - 输入编辑区（命令与提问）
  - 状态行（模型、token、模式、主题）
- 高效交互
  - 命令面板：模糊检索 + 快捷执行
  - 快速模型切换：分组检索 + 回车确认
  - 主题实时预览：列表与预览联动
- 用户引导
  - 首次启动 Onboarding 主链路（步骤化配置）
  - 进度指示、跳过入口、错误提示与恢复
- 安全防误触
  - 高风险命令意图识别
  - 二次确认输入（CONFIRM）
  - 冷却期机制（默认 3 秒）
  - 审计日志落盘

## 3. 设计规范对齐

- 布局：清晰信息层级，主任务路径固定
- 色彩：语义色统一（成功/警告/错误/信息）
- 可读性：卡片化消息、角色区分、状态可见
- 响应反馈：Toast、状态行、处理中指示器
- 响应式：窄屏自动收敛侧栏，保证主区可用

## 4. 稳定性与兼容性

- 运行模型：Textual + Rich，兼容常见终端环境
- 交互模式：全键盘可达，支持方向键与快捷键
- 性能策略：
  - 消息列表上限裁剪（防止长会话性能退化）
  - 轻量弹层与按需渲染
  - 常规路径不引入额外高频阻塞

## 5. 代码落点

- 主应用：`xiaotie/tui/app.py`
- 组件层：`xiaotie/tui/widgets.py`
- 引导系统：`xiaotie/tui/onboarding.py`
- 主题系统：`xiaotie/tui/themes.py`
- 入口：`xiaotie/tui/main.py`

## 6. 验收结论

- 满足“终端原生 + 键盘优先 + 清晰交互 + 稳定运行”目标
- 满足“可引导 + 低误触”设计目标
- 已具备持续扩展能力（主题、命令、引导步骤可扩展）
