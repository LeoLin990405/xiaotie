# 小铁 v0.11.0 迭代计划

> 基于 v0.10.0 完成的基础，综合 Kimi/DeepSeek/Qwen 建议

---

## 版本目标

**v0.11.0 - 快、稳、连**

核心目标（来自 Kimi）：
1. **快** - 响应快，优化延迟
2. **稳** - 运行稳，完善监控
3. **连** - 与常用工具连接顺畅

---

## 功能规划

### P0 - 技术债务清理

#### 1. 统一日志系统

**目标**: 结构化日志，便于排查问题

```python
# 目标 API 设计
from xiaotie import Logger, LogConfig

logger = Logger(
    config=LogConfig(
        level="INFO",
        format="json",  # json/text
        output=["console", "file"],
        file_path="~/.xiaotie/logs/",
        rotation="daily",
    )
)
```

**实现要点**:
- 使用 structlog 或 loguru
- JSON 格式便于日志分析
- 日志轮转防止磁盘占满
- 敏感信息脱敏

**复杂度**: 低 | **收益**: 高

#### 2. 性能监控指标

**目标**: 实时监控系统性能（来自 DeepSeek）

```python
# 目标 API 设计
from xiaotie import Metrics

metrics = Metrics(
    enabled=True,
    export=["prometheus", "console"],
)

# 关键指标
# - P99 延迟 < 200ms
# - 内存使用率 < 70%
# - 并发吞吐 1000qps
```

**实现要点**:
- 请求延迟统计 (P50/P95/P99)
- Token 使用统计
- 缓存命中率
- 工具执行耗时

**复杂度**: 中 | **收益**: 高

#### 3. 内存优化（来自 DeepSeek）

**目标**: 减少内存占用，防止泄漏

**实现要点**:
- 对象池技术 (Agent/Tool 实例复用)
- 流式内存管理 (大输出分块处理)
- 惰性加载 (工具按需加载)
- 会话历史压缩

**复杂度**: 中 | **收益**: 高

### P1 - 用户体验提升（来自 Kimi）

#### 4. 统一反馈机制

**目标**: 所有操作 500ms 内给出视觉反馈

```python
# 目标 API 设计
from xiaotie.tui import Feedback

feedback = Feedback(
    timeout=500,  # ms
    types=["spinner", "progress", "toast"],
)
```

**实现要点**:
- 操作状态指示器
- 进度条显示
- Toast 通知增强
- 避免用户重复操作

**复杂度**: 低 | **收益**: 中

#### 5. 上下文帮助系统

**目标**: `/?` 或 `F1` 快速呼出相关帮助

```python
# 目标 API 设计
from xiaotie import ContextHelp

help_system = ContextHelp(
    sources=["builtin", "custom"],
    search_enabled=True,
)
```

**复杂度**: 低 | **收益**: 中

#### 6. 快捷命令面板增强

**目标**: Cmd+K 快速跳转/执行常用操作

**实现要点**:
- 模糊搜索优化
- 最近使用记录
- 自定义命令
- 快捷键提示

**复杂度**: 低 | **收益**: 高

### P3 - 架构优化（来自 Qwen）

#### 9. 统一 Provider 适配层

**目标**: 消除"双轨制"，统一接口

```
xiaotie/
├── providers/
│   ├── __init__.py
│   ├── base.py          # BaseProvider 抽象类
│   ├── factory.py       # Provider 工厂
│   └── impl/
│       ├── anthropic.py
│       ├── openai.py
│       ├── glm.py
│       └── ...
```

**收益**: 统一接口、减少重复代码、便于新增 Provider

**复杂度**: 中 | **收益**: 高

#### 10. 事件总线解耦

**目标**: 组件间通过事件通信，降低耦合

```python
# 目标 API 设计
from xiaotie import EventBus, Event

bus = EventBus()

# 事件流
# RequestReceived → RouterDecided → ProviderCalled → ResponseReceived
```

**收益**: 各组件独立演进、支持异步削峰、易于扩展

**复杂度**: 中 | **收益**: 中
